# Публикация «Где 95-й» в Google Play

Приложение = PWA, обёрнутая в **TWA** (Trusted Web Activity). На выходе — обычный
`.aab`, который принимает Play Console. Контент при этом обновляется на сервере,
а не через магазин.

## Схема

```
[ конвейер ] → cities.json → [ хостинг HTTPS ] ← тянет → [ Android-приложение (TWA) ]
                                     ↑
                          assetlinks.json (связка домена и приложения)
```

---

## Шаг 1. Захостить папку `public/` на HTTPS

Нужен домен под HTTPS (TWA работает только по HTTPS и требует своего домена, чтобы
положить `assetlinks.json`). Варианты:

- **GitHub Pages** (бесплатно, рекомендую для старта): репозиторий → Settings → Pages
  → отдавать папку `public/`. Адрес вида `https://<user>.github.io/gde-95/`.
  Обновление данных = `git push`.
- **Cloudflare Pages / Netlify** — тоже бесплатно, свой домен, деплой из гита.
- **Свой сервер** (у нас уже есть под другие сервисы) — раздать `public/` статикой
  по HTTPS, обновлять `cities.json` тем же конвейером.

После хостинга проверь, что открываются:
- `https://ДОМЕН/` — приложение
- `https://ДОМЕН/manifest.webmanifest`
- `https://ДОМЕН/data/cities.json`
- `https://ДОМЕН/.well-known/assetlinks.json`

## Шаг 2. Аккаунт разработчика Google Play

Разовый взнос **$25**. Регистрация: https://play.google.com/console/signup
(нужен Google-аккаунт и карта). Без этого опубликовать в стор нельзя — это
единственный ручной внешний шаг.

## Шаг 3. Собрать `.aab` через Bubblewrap

Нужны **JDK 17** и **Android SDK** (Bubblewrap умеет доставить их сам при `init`).

```bash
npm i -g @bubblewrap/cli

# в папке twa/ поправить twa-manifest.json: заменить PLACEHOLDER.github.io на свой домен
cd twa
bubblewrap init --manifest https://ДОМЕН/manifest.webmanifest   # или используй готовый twa-manifest.json
bubblewrap build       # создаст app-release-signed.aab + подписной ключ
```

При первой сборке Bubblewrap создаст ключ подписи (`android.keystore`) — **сохрани его
и пароль** (без него нельзя выпускать обновления). Лучше включить Play App Signing.

## Шаг 4. Прописать связку домена и приложения (assetlinks)

1. В Play Console → Release → Setup → App integrity скопируй **SHA-256** отпечаток
   ключа подписи приложения.
2. Вставь его в `public/.well-known/assetlinks.json` вместо
   `ЗАМЕНИ_НА_SHA256_...`, задеплой сайт заново.
3. Проверка: `https://ДОМЕН/.well-known/assetlinks.json` отдаёт JSON с твоим SHA-256.

Без этого шага приложение откроется, но с адресной строкой браузера сверху.
С ним — полноэкранный «настоящий» вид.

## Шаг 5. Загрузить в Play Console

1. Создать приложение → язык русский → бесплатное.
2. Загрузить `.aab` в трек (сначала **внутреннее тестирование**, потом прод).
3. Заполнить карточку стора (см. `docs/STORE-LISTING.md`): описание, скриншоты
   (телефон, мин. 2), иконка 512×512 (`public/icons/icon-512.png`), фичер-графика
   1024×500.
4. Заполнить контент-рейтинг, политику конфиденциальности (нужен URL), данные о
   безопасности данных (приложение НЕ собирает персональные данные — это плюс к ревью).
5. Отправить на проверку. Первое ревью обычно от нескольких часов до пары дней.

## Обновления после релиза

- **Данные** (города, цены, лимиты) — обновляются на хостинге конвейером, в сторе
  ничего делать не нужно, у пользователя всё свежее при следующем открытии.
- **Само приложение** (экран, логика) — пересобрать `.aab` с новым `appVersionCode`
  и загрузить в Play Console.

## Быстрый путь без стора (пока идёт регистрация)

PWA ставится на телефон и без магазина: открыть сайт в Chrome на Android → меню →
«Установить приложение». Иконка появится на экране, работает офлайн. Это удобно,
чтобы показать/раздать ссылкой уже сегодня, параллельно с публикацией в Google Play.
